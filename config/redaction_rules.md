# 脱敏规则：研发/市场资料进入 deep-research 前使用

## 必须脱敏的信息

- 客户名称；
- 项目代号；
- 内部芯片型号；
- 未发布产品规划；
- 报价、毛利、成本；
- 合同条款；
- 供应商未公开信息；
- 量产时间表；
- 客户邮件、微信、联系人信息。

## 推荐替换格式

| 原始信息 | 替换格式 |
|---|---|
| 客户公司名 | 客户A、客户B |
| 项目名 | 项目P1、项目P2 |
| 内部芯片型号 | 芯片X1、芯片X2 |
| 价格 | 价格区间A、价格区间B |
| 联系人 | 联系人R1、联系人R2 |
| 具体日期 | 2026Q1、2026Q2 等季度粒度 |

## 联网模式硬规则

只要主题或资料中包含上述敏感信息，就不要使用 `/research-public`。

应改用：

```text
/research-local <本地路径> <脱敏后的主题>
```

## 允许进入公开模式的信息

- 已发布 datasheet；
- 官网公开新闻；
- 公开招聘信息；
- 公开专利；
- 公开财报；
- 公开行业报告；
- 已公开标准或法规。
